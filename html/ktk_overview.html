An overview of Kinetics Toolkit#

Kinetics Toolkit is designed to play well with other softwares and packages, to be used as an addition and not as a replacement to existing workflows. Most of the data processing is done via the ktk.TimeSeries class, which is a container for time (np.array), data (Dict[np.array]), events and metadata. This tutorial on TimeSeries will help to understand this class and its associated methods.

Getting your data in#

(orange in figure below)

Kinetics Toolkit does not ties to specific instruments. Using your recording instrumentation, you need to get your data in a compatible file, be it very generic such as csv or txt files, or more specialized such as c3d.

Any data that can be read as a Pandas DataFrame (e.g., csv, txt, Excel, SQL) can be converted to a TimeSeries using ktk.TimeSeries.from_dataframe. This tutorial shows an example. To import motion data from a c3d file, use the ktk.read_c3d function.

In addition to importing and exporting, Kinetics Toolkit offers ktk.load and ktk.save functions, which store standard python types, NumPy arrays, basic Pandasâ€™ DataFrames and Series, and ktkâ€™s TimeSeries in a portable zipped json format. See this tutorial for more information.

Processing and visualizing your data#

(blue in figure above)

Once your data loaded as TimeSeries, you can use:

  • the TimeSeries methods to add and edit events, to sync, split, subset, merge, resample TimeSeries, etc. (tutorials)

  • the ktk.filters module to filter data using butterworth filters, smoothing filters, deriving filters, etc. (tutorials);

  • the ktk.cycles module to detect, time-normalize, combine cycles, find most repeatable cycles, etc. (tutorials);

  • the ktk.geometry module to create rigid transforms, frames, convert between local and global coordinates, extract angles, etc. (tutorials);

  • the ktk.kinematics module to create and track marker clusters, reconstruct virtual markers, etc. (tutorials);

  • the ktk.Player class to visualize animated, interconnected markers and frames in an interactive 3D environment (tutorial);

  • the ktk.TimeSeries.plot method to plot 2D graphes of your TimeSeries contents, including the events.

If applicable, you can also use your current functions:

  • that use NumPy arrays: by working directly on the TimeSeriesâ€™ data attribute;

  • that use Pandas DataFrames: using ktk.TimeSeries.from_dataframe and ktk.TimeSeries.to_dataframe.

Getting your data out#

(red in figure above)

Once your data are processed, you can export it to other formats to process it using other software, using a similar method that you took to import those in a first place: use ktk.write_c3d for c3d, or Pandasâ€™ to_ methods for saving csv, txt, Excel, etc., via the ktk.TimeSeries.to_dataframe method. You can also save your variables using ktk.save for short-term to long-term storage.
